# Contributing to AgriFlux

We love your input! We want to make contributing to this project as easy and transparent as possible.

## Development Setup
1. Clone the repo and navigate to `Frontend` and `Backend`.
2. Run `npm install` in both directories.
3. Configure `.env` in the backend (use `.env.example`).
4. Run `npm run dev` to start both servers.

## Pull Requests
- Keep PRs focused on a single feature or bugfix.
- Ensure TypeScript compiles without errors.
- Adhere to the existing ESLint and code style guidelines.
- Add clear descriptions and screenshots for UI changes.

## Code Structure
- `/Frontend/src/pages`: Page-level components
- `/Frontend/src/components`: Reusable UI elements
- `/Backend/src/controllers`: Request/Response logic
- `/Backend/src/routes`: API endpoint definitions

Please open an issue to discuss major changes before submitting a PR.
